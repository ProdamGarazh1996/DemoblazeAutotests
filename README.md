# 🚀 Автоматизированное тестирование demoblaze.com

Проект предназначен для автоматизированного тестирования функциональности сайта **[demoblaze.com](https://demoblaze.com)**. Репозиторий имеет модульную структуру, разделяющую UI и API тесты, и полностью готов к запуску как локально, так и в CI/CD пайплайнах.

## 🛠 Стек технологий

*   **Язык программирования**: Java 21
*   **Сборщик проектов**: Maven 3
*   **Тестовый фреймворк**: TestNG
*   **Тестирование API**: Rest Assured
*   **CI/CD инструмент**: Jenkins

## 📂 Структура проекта

Проект является многомодульным (Multi-module Maven Project) и состоит из двух основных изолированных частей:
*   `ui-tests` — модуль, содержащий автотесты для пользовательского интерфейса (UI).
*   `api-tests` — модуль, содержащий автотесты для REST API интерфейсов приложения.

## 📥 Требования для локального запуска

Перед запуском убедитесь, что у вас установлены:
1.  **JDK 21** (с настроенной переменной окружения `JAVA_HOME`).
2.  **Maven 3** (с настроенной переменной окружения `M2_HOME` и добавленным `mvn` в PATH).
3.  Браузеры **Google Chrome** и/или **Mozilla Firefox** (для UI-модуля).

## 🚀 Команды для запуска тестов

### 1. Запуск отдельных модулей

*   **Только UI тесты:**
    ```bash
    mvn clean test -pl ui-tests -am
    ```
*   **Только API тесты:**
    ```bash
    mvn clean test -pl api-tests -am
    ```
*   **Все тесты (UI + API одновременно):**
    ```bash
    mvn clean test -pl api-tests,ui-tests -am
    ```

### 2. Параметры конфигурации запуска

Запуск можно гибко настраивать с помощью системных свойств Maven (`-D`):

*   `-Dgroups=<имя_группы>` — запуск тестов, принадлежащих к определенной группе TestNG (например, `smoke`, `regression`).
*   `-Dbrowser=<браузер>` — выбор браузера для UI тестов. Поддерживаемые значения: `chrome`, `firefox`.
*   `-Dlogin=<логин>` — передача учетных данных пользователя (Логин).
*   `-Dpass=<пароль>` — передача учетных данных пользователя (Пароль).

**Пример комплексной команды запуска:**
```bash
mvn clean test -pl ui-tests -am -Dbrowser=chrome -Dgroups=smoke -Dlogin=myUser -Dpass=mySecretPassword
```

## 🏗 Интеграция с CI/CD (Jenkins)

Проект полностью автоматизирован для запуска внутри **Jenkins** с использованием декларативного пайплайна (**Jenkinsfile**). Сборка поддерживает параметризацию, автоматическое расписание и генерацию отчетов **Allure Report**.

### ⚙️ Требования к окружению Jenkins
Для успешного выполнения пайплайна в Global Tool Configuration Jenkins должны быть настроены следующие инструменты:
*   **Maven** с именем `Maven_3`
*   **JDK** с именем `Java_21`
*   **Allure Commandline** с именем `Allure`
*   Плагин **Parameterized Cron** (для поддержки расписания с параметрами)

### 🎛 Параметры сборки (Pipeline Parameters)
При ручном запуске пайплайна доступны следующие настройки:
*   `BRANCH_NAME` — ветка Git для скачивания кода (по умолчанию `main`).
*   `TEST_GROUPS` — группа тестов TestNG для запуска (`all`, `regression`, `smoke`).
*   `MODULE` — целевой модуль для тестирования (`all`, `ui-tests`, `api-tests`).
*   `BROWSER` — браузер для UI-тестов (`firefox`, `chrome`). При выборе `api-tests` параметр игнорируется.

### ⏱ Автоматический запуск (Расписание)
В пайплайне настроен периодический запуск тестов по расписанию с помощью `parameterizedCron`:
*   **Каждые 15 минут** запускаются UI-тесты в браузере **Firefox** на ветке `main`.
*   **Каждые 20 минут** запускаются только API-тесты на ветке `main`.

### 🔐 Безопасность и учетные данные
Логин и пароль для авторизации на сайте `demoblaze.com` не хранятся в открытом виде. Пайплайн безопасно извлекает их из хранилища Jenkins Credentials по идентификатору `demoblaze-user` и прокидывает в переменные окружения `%USR%` и `%PSW%` во время выполнения Maven-команды на Windows-агенте (`bat`).

### 📊 Отчетность (Allure Report)
После завершения тестов (в блоке `post { always }`) плагин Allure автоматически собирает результаты из целевых директорий обоих модулей:
*   `ui-tests/target/allure-results`
*   `api-tests/target/allure-results`

Сгенерированный отчет прикрепляется к текущей сборке в интерфейсе Jenkins.
